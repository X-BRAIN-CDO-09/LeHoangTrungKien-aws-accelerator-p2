K8s on AWS - Terraform 1-Click

Architecture Flow

Tài liệu này mô tả hai luồng chính của lab: cách user truy cập app qua ALB, và cách Terraform deploy hạ tầng + image + Kubernetes app.

Luồng User Truy Cập Hệ Thống

User

Browser

User mở URL public lấy từ output alb_url.

AWS

ALB :80

Application Load Balancer nhận HTTP request từ Internet.

AWS

Target Group

ALB forward request vào EC2 target qua port 30080.

K8s

kind NodePort

EC2 hostPort 30080 được map vào Service nodePort: 30080.

Pod

Demo App :80

Service chuyển traffic tới Pod chạy static app bằng nginx.

Port cố định

Biến app_node_port mặc định là 30080, nằm trong dải NodePort hợp lệ 30000-32767.

Security Group

ALB mở port 80 từ Internet. EC2 chỉ nhận 30080 từ security group của ALB.

App không chạy trực tiếp trên EC2

EC2 chỉ là host chạy Docker và kind. App thật sự chạy trong Kubernetes Pod.

Luồng Terraform Deploy

Terraform

Providers

Dùng hashicorp/aws để tạo AWS resources và hashicorp/tls, hashicorp/local, hashicorp/cloudinit cho key pair, file key local và user data.

ECR

Image Registry

Terraform tạo ECR repository để lưu image của demo app.

Local

Build & Push

Máy local build Docker image từ demo-app rồi push lên ECR.

AWS

Infra

Terraform tạo VPC, subnets, security groups, EC2, IAM role, ALB và target group.

Bootstrap

user_data

EC2 cài Docker, kubectl, kind, tạo cluster và apply Deployment + Service.

Thứ tự đảm bảo

EC2 phụ thuộc vào bước image build/push, nên khi bootstrap chạy thì image đã có sẵn trên ECR.

Kubernetes pull image

EC2 dùng IAM role để lấy ECR token, tạo image pull secret và để Pod pull image từ ECR.

Output kiểm tra

Sau khi apply, dùng terraform output alb_url để lấy URL public của app.
